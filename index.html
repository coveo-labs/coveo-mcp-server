Core Features

Built with performance and developer experience in mind.

Asynchronous by Design

Leverages `httpx` and `asyncio` for non-blocking API requests, ensuring a responsive server.

Grounded Generative Answering

Go beyond search results with AI-generated answers complete with citations from your indexed content.

Semantic Retrieval

The `passage_retrieval` tool intelligently extracts the most relevant text snippets, not just whole documents.

Easy Configuration

Simple setup using a `.env` file for your Coveo credentials. Get up and running in minutes.

Multiple Transports

Supports modern `streamable-http`, legacy `SSE`, and direct `STDIO` for maximum client compatibility.

Fully Tested

Includes a comprehensive test suite with `pytest` to ensure reliability and correctness.

Get Started in 4 Steps

Go from zero to a running Coveo MCP server in minutes.

Step 1: Clone the Repository

Get the source code on your local machine.

git clone https://github.com/coveo-labs/coveo-mcp-server.git
cd coveo-mcp-server

Step 2: Configure Your Environment

Copy the example environment file and add your Coveo API credentials.

cp .env.example .env

Then, edit `.env`:

COVEO_API_KEY="<your-api-key>"
COVEO_ORGANIZATION_ID="<your-organization-id>"
COVEO_ANSWER_CONFIG_ID="<your-answer-config-id>"

Step 3: Install Dependencies

We recommend using `uv` for fast dependency management.

uv pip install -e '.[dev]'

Step 4: Run the Server

Start the server. By default, it uses the modern `streamable-http` transport.

python -m coveo_mcp_server

You can test it immediately with the MCP Inspector:

mcp dev src/coveo_mcp_server/__main__.py
